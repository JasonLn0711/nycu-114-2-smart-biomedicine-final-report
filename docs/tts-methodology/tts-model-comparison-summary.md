# TTS Model Comparison Summary

This table summarizes the Smart Biomedicine final-report TTS attempts and turns
them into reusable evidence for future research-audio production.

| model | strength | weakness | accepted use | rejected use | next test |
| --- | --- | --- | --- | --- | --- |
| BreezyVoice 26 / `MediaTek-Research/BreezyVoice-300M` | Produced the accepted final narration route after chunk-level repair, ASR back-transcription, semantic/critical phrase checks, stitching, loudness QA, and provenance capture. The v0.1 auto-QA run reports `accepted_with_warnings`, `CER=0.0087`, `WER=0.0175`, and `critical_term_accuracy=1.0000`. | English dense noun phrases, acronyms, negation, and clinical-boundary wording required repeated model-facing text repair. Current warnings are delivery/platform smoke, near-ceiling audio peak, synthetic-voice disclosure, and sentence heuristic watch. | Final Smart Biomedicine narration package and future long-form research narration when chunked and gated. | Ungated one-pass English long-form generation; stale-source audio; CPU fallback. | Run the golden pilot set before the next new model comparison. |
| F5-TTS `F5TTS_v1_Base` | Generated all `14` chunks on RTX 5080 with a clear manifest and repeatable command. Fast candidate route for pilot voice generation. | The generated master was shorter than the accepted BreezyVoice target and did not yet pass the same ASR/term/audio/chunk gates. | Future pilot clips and model comparison baselines. | Final research package before ASR back-transcription and audio QA are added. | Run the same automated QA rubric and compare CER/WER, critical-term accuracy, loudness, and chunk consistency against BreezyVoice. |
| GPT-SoVITS | Produced detailed repair history and many useful failure patterns for text design, ASR gate design, critical phrase lists, and pronunciation instability. | Historical accepted markers do not transfer after a model switch; earlier flow depended on manual semantic sweep and did not represent the final route. | Failure taxonomy, text-repair rules, pronunciation lexicon seeds, and historical comparison. | Final package after switching to BreezyVoice; any route without current-source provenance. | Re-run a small pilot under the new automated rubric only if a future project needs a GPT-SoVITS baseline. |
| CosyVoice 2 / CosyVoice 3 | Web-verified on `2026-05-31` as an active multilingual zero-shot / streaming TTS family with official GitHub, papers, and Fun-CosyVoice3 Hugging Face model route. It was not part of this Smart Biomedicine production run. | No local output, no ASR transcript, no term check, no audio QA, no rights manifest, and no GPU/runtime evidence in this repo yet. | Future golden-pilot comparison, especially if multilingual, low-latency, streaming, or in-the-wild speech robustness matters. | Any final research or teaching package before local pilot evidence exists. | Add CosyVoice 3 to the next golden-pilot run and compare against BreezyVoice / F5-TTS under the same auto-QA rubric. |
| OpenAI Audio API / `gpt-4o-mini-tts` | Web-verified on `2026-05-31` as a current cloud TTS route through the OpenAI Audio API, with official disclosure requirements for AI-generated speech. It was not part of this Smart Biomedicine production run. | Not local or open-weight; reproducibility depends on model snapshot, request parameters, API policy, data-retention configuration, and network/service state. No local audio or QA evidence exists in this repo. | Optional cloud baseline for speed, multilingual delivery, or product-style narration when governance permits cloud processing. | Any sensitive reference-voice, patient-like, or research-stimulus route without data-governance approval, snapshot pinning, disclosure, and full QA output. | If used, record model snapshot, request JSON, response hash, disclosure text, data boundary, and second-ASR audit before comparing with local models. |

## Decision

Use BreezyVoice-style chunked generation plus `tools/run_tts_auto_qa.py` as the
current reference workflow. Treat F5-TTS and GPT-SoVITS as candidate or
historical routes until they pass the same automated gates and golden pilot set.
Treat CosyVoice 3 and OpenAI Audio API as web-verified candidates to add to the
golden-pilot queue, not as accepted routes.
